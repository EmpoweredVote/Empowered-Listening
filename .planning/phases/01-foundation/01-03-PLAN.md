  DATABASE_TRANSACTION_POOLER_URL: z.string().optional(),
  AUTH_BYPASS: z.enum(['0', '1']).optional(),
  NODE_ENV: z.enum(['development', 'production', 'test']).default('development'),
});

export const env = schema.parse(process.env);

// Safety check: refuse to start if AUTH_BYPASS=1 in non-dev (locked decision from CONTEXT.md)

  );
}

  <how-to-verify>
**Step 1 — Authenticate Wrangler (one-time):**
```bash
npx wrangler login
```
A browser window opens; approve the Wrangler OAuth.  Confirm success by running `npx wrangler whoami` — expect your Cloudflare account email.

**Step 2 — Paste secrets into .dev.vars:**
Copy the values from your password manager (populated in plan 01-02) into `.dev.vars`.  At minimum for this deploy, you need:
- `NEXT_PUBLIC_SUPABASE_URL` (already known: https://kxsdzaojfaibhuzmclfq.supabase.co)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` (from Supabase dashboard -> Settings -> API)
- `ACCOUNTS_API_BASE=https://api.empowered.vote`
- `JWKS_URL=https://kxsdzaojfaibhuzmclfq.supabase.co/auth/v1/.well-known/jwks.json`

Rest are not needed for the holding-page deploy but can be populated now.

**Step 3 — Build and deploy:**
```bash
npm run build
npm run deploy
```

This runs OpenNext's build then Wrangler's deploy.  Expect a URL like `https://empowered-listening.<your-subdomain>.workers.dev` to be printed at the end.  Visit it — the holding page should render.

**Step 4 — Sync non-public env vars to the Worker:**
For each server-only secret in `.dev.vars`, push it to the Worker:
```bash
npx wrangler secret put NEXT_PUBLIC_SUPABASE_ANON_KEY
# paste the value when prompted; repeat for any other server secrets
```

`NEXT_PUBLIC_*` vars are inlined at build time and don't need `wrangler secret put` — set them via the dashboard as "Variable" (plaintext), or ensure they're in `.dev.vars` at build time.  (OpenNext reads `.dev.vars` for the build; confirm by checking `.open-next/` output.)

**Step 5 — Bind custom domain (browser, Cloudflare dashboard):**
1. Go to Cloudflare Dashboard -> Workers & Pages -> empowered-listening -> Settings -> Domains & Routes.
2. Click "Add -> Custom Domain".
3. Enter `listening.empowered.vote`.  Cloudflare auto-creates the DNS record (you must own the `empowered.vote` zone in this same account, which was confirmed in plan 01-02).
4. Wait for the TLS cert to issue (usually 30 seconds).

**Step 6 — Verify the live deploy:**
1. Visit https://listening.empowered.vote — expect the holding page to render identically to localhost, with valid HTTPS.
2. Visit https://listening.empowered.vote/api/health — expect 200 JSON.

5. Confirm no console errors in the browser DevTools.
