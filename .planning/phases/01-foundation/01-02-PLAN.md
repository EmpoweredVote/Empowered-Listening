<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Provision LiveKit Cloud, Cloudflare Stream + R2, and Deepgram accounts</name>
  <what-built>
These services have no CLI that can create an account from scratch — account creation, plan selection (Stream is paid), and initial API key generation all happen in browser dashboards.  Claude can help script the work once credentials exist, but sign-up is inherently human-operator work.
  </what-built>
  <how-to-verify>
Complete all three provisioning stops, in any order.  Mark each as done in the checklist file (created in Task 2) as you finish.

**Stop 1 — LiveKit Cloud (5-10 min)**
1. Visit https://cloud.livekit.io and sign up or log in (GitHub sign-in is simplest).
2. Click "New Project".  Name it `empowered-listening`.  Choose the region closest to Indiana (likely `us-east`).
3. From the project sidebar, copy:
   - **WebSocket URL** (appears as `wss://empowered-listening-xxxxx.livekit.cloud`) -> save as `LIVEKIT_URL`
4. Go to Keys -> "Add New Key".  Label it `empowered-listening-server`.  Copy:
   - **API Key** -> save as `LIVEKIT_API_KEY`
   - **API Secret** (shown ONCE at creation — store immediately) -> save as `LIVEKIT_API_SECRET`
5. Confirm the free tier quota (LiveKit Cloud free tier covers early pilot usage; no billing card required to start).

**Stop 2 — Cloudflare Stream + R2 (10-15 min)**
1. Log in to the Cloudflare account that owns `empowered.vote` (Chris's primary EV Cloudflare account — confirm before proceeding).
2. Copy the Account ID from the right sidebar of Workers & Pages -> save as `CLOUDFLARE_ACCOUNT_ID`.
3. **Stream** — Go to Stream from the left nav.  If it says "Subscribe", subscribe at $5/mo (minimum tier; pay-as-you-go per minute of delivered video).  Once active, go to Settings and copy:
   - **Customer subdomain** (format: `customer-xxxxxxxxxxxxx.cloudflarestream.com`) -> save as `CLOUDFLARE_STREAM_CUSTOMER_SUBDOMAIN`
4. **R2** — Go to R2 Object Storage.  If not enabled, enable it (free tier covers 10 GB storage, Class A/B operations — easily fits early pilot).
5. Click "Create bucket".  Name: `empowered-listening-recordings`.  Location: Automatic.  Save bucket name as `CLOUDFLARE_R2_BUCKET_NAME`.
6. Go to "Manage R2 API Tokens" -> "Create API token".  Token name: `empowered-listening-recordings-rw`.  Permissions: Object Read & Write.  Specify bucket: `empowered-listening-recordings`.  TTL: no expiration (or 1 year; document the rotation date).
7. Copy:
   - **Access Key ID** -> save as `CLOUDFLARE_R2_ACCESS_KEY_ID`
   - **Secret Access Key** (shown ONCE) -> save as `CLOUDFLARE_R2_SECRET_ACCESS_KEY`

**Stop 3 — Deepgram (5 min)**
1. Sign up at https://console.deepgram.com (free tier includes $200 of credit — plenty for pilot transcription).
2. Go to API Keys -> "Create a Key".  Name: `empowered-listening-transcription`.  Scope: `Member` (NOT Admin — least privilege).
3. Copy:
   - **API Key** -> save as `DEEPGRAM_API_KEY`

Store all copied values in the operator's password manager or a secure team secrets store.  NEVER paste into a file committed to git.  They will be pasted into `.dev.vars` (gitignored) during plan 01-03.
  </how-to-verify>
