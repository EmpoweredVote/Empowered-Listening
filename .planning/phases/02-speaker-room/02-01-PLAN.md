<objective>
Extend the `listening` schema with the columns Phase 2 needs for server-authoritative timers, prep time pools, and faithful pause/resume (paused_remaining_seconds), enable Supabase Realtime for the three tables clients will subscribe to, amend RLS so debate participants can read their scheduled debate before it goes live, and create the SECURITY DEFINER RPCs that segment transitions and prep-time control will call.  All state-transition writes to the `listening` schema will go through these RPCs so the server can enforce atomicity (segments + debates + debate_speakers updated together), moderator-identity gating, and the fixed-sequence order required by DEBATE-04.

Purpose: Phase 1 delivered the schema but did not include `end_time`, `prep_time_end_time`, `paused_remaining_seconds`, or `prep_time_seconds` — all four are required for Phase 2.  Phase 1 RLS only allowed SELECT on `live` / `completed` debates; speakers joining a still-`scheduled` debate would be blocked from snapshot fetch, Realtime subscription, and token mint — this plan extends RLS to let debate participants read their own scheduled debate regardless of status.  Supabase Realtime does not fire `postgres_changes` events for non-public schemas without explicit `GRANT SELECT` + `REPLICA IDENTITY FULL` + publication additions.  Getting all of this into one migration pair keeps Phase 2 app code focused on business logic instead of schema plumbing.

The first adds the missing columns, enables Realtime for three tables, and adds participant-visibility RLS policies for scheduled debates.  The second defines the RPC functions for segment start / end / repeat (with server-enforced sequence order), prep-time start / end (with faithful pause/resume via paused_remaining_seconds), and bonus-time consumption.  After this plan ships, Phase 2 app code can `await supabaseAdmin.rpc('start_segment', {...})`, subscribe to postgres_changes on `listening.debate_segments`, and rely on the DB to reject out-of-order segment starts.

-- Phase: 02-speaker-room / Plan: 02-01
-- Applied to: Supabase project kxsdzaojfaibhuzmclfq
--
-- Adds columns required for server-authoritative timers (end_time, prep_time_end_time),
-- faithful pause/resume (paused_remaining_seconds), and per-speaker prep time pool tracking
-- (prep_time_seconds default 240 = 4 minutes).
--
-- Enables Supabase Realtime postgres_changes broadcasting for the tables that Phase 2
-- clients subscribe to.  Non-public schemas need GRANT SELECT + REPLICA IDENTITY FULL +
-- explicit publication add; public-schema tables get these automatically.
--
-- Amends RLS: Phase 1 policies only allow SELECT on live/completed debates.  Speakers
-- joining a still-scheduled debate need visibility via their debate_speakers row.

-- ============================================================
-- 1. Column additions
-- ============================================================

-- Server-authoritative segment timer — stores the absolute time at which the current
-- segment's allocated_seconds runs out.  Clients compute remaining = end_time - Date.now()
-- on their local tick.  Null when segment is upcoming or completed, or paused (see below).
ALTER TABLE listening.debate_segments
  ADD COLUMN IF NOT EXISTS end_time timestamptz;

-- Server-authoritative prep-time pause timer.  When a speaker calls for prep time during
-- their segment, the main segment timer pauses and this column records the absolute time
-- at which prep runs out.  Set by listening.start_prep_time RPC; cleared by listening.end_prep_time.
ALTER TABLE listening.debate_segments
  ADD COLUMN IF NOT EXISTS prep_time_end_time timestamptz;

-- Snapshot of the main segment timer's remaining seconds at the moment prep time was started.
-- When prep ends, end_time is restored as NOW() + paused_remaining_seconds (faithful pause/resume,
-- NOT a full reset).  NULL when the segment is not currently in prep-pause state.
-- This is the BLOCKER 1 fix: without it, end_prep_time reset to full allocated duration.
ALTER TABLE listening.debate_segments
  ADD COLUMN IF NOT EXISTS paused_remaining_seconds integer;

-- Per-speaker prep time pool.  Standard LD format: 4 minutes per speaker.  Decrements
-- as the speaker consumes prep time during any of their segments.  Does not replenish.
ALTER TABLE listening.debate_speakers
  ADD COLUMN IF NOT EXISTS prep_time_seconds integer NOT NULL DEFAULT 240;

-- ============================================================
-- 2. Realtime enablement for Phase 2 subscriptions
-- ============================================================
-- The `listening` schema is non-public.  Supabase Realtime requires:
--   (a) GRANT SELECT to authenticated so the WAL listener can filter rows per user
--   (b) REPLICA IDENTITY FULL so updates broadcast full row payloads (needed to filter
--       on non-PK columns like debate_id)
--   (c) explicit addition to the supabase_realtime publication (public schema tables get
--       this automatically; non-public schemas do not)
--
-- GRANTs here are necessary in addition to the RLS policies below — RLS is applied on top.

GRANT SELECT ON listening.debates TO authenticated;
GRANT SELECT ON listening.debate_segments TO authenticated;
GRANT SELECT ON listening.debate_speakers TO authenticated;

ALTER TABLE listening.debates REPLICA IDENTITY FULL;
ALTER TABLE listening.debate_segments REPLICA IDENTITY FULL;
ALTER TABLE listening.debate_speakers REPLICA IDENTITY FULL;

ALTER PUBLICATION supabase_realtime ADD TABLE listening.debates;
ALTER PUBLICATION supabase_realtime ADD TABLE listening.debate_segments;
ALTER PUBLICATION supabase_realtime ADD TABLE listening.debate_speakers;

-- ============================================================
-- 3. RLS amendments — participant visibility for scheduled debates
-- ============================================================
-- BLOCKER 5 fix: Phase 1 RLS policies only permit SELECT on debates with
-- status IN ('live', 'completed').  Speakers joining through their share
-- link need to read the debate while it is still `scheduled` — otherwise
-- the snapshot fetch, Realtime subscription, and token-mint endpoint all fail.
--
-- Strategy: add (or replace) a SELECT policy that UNIONs the public visibility
-- with participant visibility via listening.debate_speakers.  We drop the old
-- policy names first if they exist so this migration is idempotent across re-runs.

-- debates
DROP POLICY IF EXISTS "debates_public_select" ON listening.debates;
DROP POLICY IF EXISTS "debate_participants_can_read_scheduled" ON listening.debates;
CREATE POLICY "debate_participants_can_read_scheduled"
  ON listening.debates FOR SELECT
  TO authenticated
  USING (
    status IN ('live', 'completed')
    OR EXISTS (
      SELECT 1 FROM listening.debate_speakers
      WHERE debate_id = debates.id AND user_id = auth.uid()
    )
  );

-- debate_segments
DROP POLICY IF EXISTS "debate_segments_public_select" ON listening.debate_segments;
DROP POLICY IF EXISTS "debate_participants_can_read_segments" ON listening.debate_segments;
CREATE POLICY "debate_participants_can_read_segments"
  ON listening.debate_segments FOR SELECT
  TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM listening.debates d
      WHERE d.id = debate_segments.debate_id
        AND (
          d.status IN ('live', 'completed')
          OR EXISTS (
            SELECT 1 FROM listening.debate_speakers ds
            WHERE ds.debate_id = d.id AND ds.user_id = auth.uid()
          )
        )
    )
  );

-- debate_speakers
DROP POLICY IF EXISTS "debate_speakers_public_select" ON listening.debate_speakers;
DROP POLICY IF EXISTS "debate_participants_can_read_speakers" ON listening.debate_speakers;
CREATE POLICY "debate_participants_can_read_speakers"
  ON listening.debate_speakers FOR SELECT
  TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM listening.debates d
      WHERE d.id = debate_speakers.debate_id
        AND (
          d.status IN ('live', 'completed')
          OR EXISTS (
            SELECT 1 FROM listening.debate_speakers ds2
            WHERE ds2.debate_id = d.id AND ds2.user_id = auth.uid()
          )
        )
    )
  );

Each verifies the caller is the debate moderator before writing (except `consume_bonus_time` which is called by the server after mic auto-mute).  `start_segment` enforces fixed sequence order (BLOCKER 3).  `start_prep_time` / `end_prep_time` use the new `paused_remaining_seconds` column for faithful pause/resume (BLOCKER 1).

```sql
-- Migration: Phase 2 speaker room state-transition RPCs
-- Phase: 02-speaker-room / Plan: 02-01
--
-- All functions are SECURITY DEFINER so they run with the schema owner's privileges,
-- bypass RLS, and update multiple tables atomically.  search_path = '' prevents
-- privilege-escalation via mutable search paths.
-- Moderator identity is verified inside each function via a listening.debate_speakers lookup.

-- ============================================================
-- 1. start_segment — activate a segment, set end_time, transition debate to live
--    ENFORCES FIXED SEQUENCE ORDER (DEBATE-04) — rejects out-of-order starts.
-- ============================================================
CREATE OR REPLACE FUNCTION listening.start_segment(
  p_debate_id uuid,
  p_segment_id uuid,
  p_moderator_user_id uuid,
  p_duration_seconds integer
)
RETURNS listening.debate_segments
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = ''
AS $$
DECLARE
  v_segment listening.debate_segments;
  v_end_time timestamptz;
  v_target_order integer;
  v_max_completed integer;
BEGIN
  -- Moderator identity check
  IF NOT EXISTS (
    SELECT 1 FROM listening.debate_speakers
    WHERE debate_id = p_debate_id AND user_id = p_moderator_user_id AND role = 'moderator'
  ) THEN
    RAISE EXCEPTION 'Not authorized: caller is not the debate moderator';
  END IF;

  -- BLOCKER 3 fix: enforce fixed sequence order.  Look up the target segment's
  -- sequence_order and the max completed sequence_order for this debate.
  SELECT sequence_order INTO v_target_order
  FROM listening.debate_segments
  WHERE id = p_segment_id AND debate_id = p_debate_id;

  IF v_target_order IS NULL THEN
    RAISE EXCEPTION 'Segment % not found for debate %', p_segment_id, p_debate_id;
  END IF;

  SELECT COALESCE(MAX(sequence_order), 0) INTO v_max_completed
  FROM listening.debate_segments
  WHERE debate_id = p_debate_id AND status = 'completed';

  -- Valid starts: (a) target is first segment AND no completed segments exist, OR
  --               (b) target is exactly max(completed) + 1.
  -- NOTE: repeat_segment is exempt — it re-activates an already-started segment and
  -- does not go through start_segment.
  IF NOT (
    (v_target_order = 1 AND v_max_completed = 0)
    OR (v_target_order = v_max_completed + 1)
  ) THEN
    RAISE EXCEPTION 'Invalid sequence: cannot start segment % out of order (max completed = %)',
      v_target_order, v_max_completed;
  END IF;

  v_end_time := NOW() + (p_duration_seconds || ' seconds')::interval;

  -- Mark any currently active segment completed (enforces single-active invariant)
  UPDATE listening.debate_segments
  SET status = 'completed', actual_end = NOW()
  WHERE debate_id = p_debate_id AND status = 'active';

  UPDATE listening.debate_segments
  SET status = 'active', actual_start = NOW(), end_time = v_end_time,
      prep_time_end_time = NULL, paused_remaining_seconds = NULL
  WHERE id = p_segment_id
  RETURNING * INTO v_segment;

  UPDATE listening.debates
  SET status = 'live', actual_start = COALESCE(actual_start, NOW())
  WHERE id = p_debate_id;

  RETURN v_segment;
END;
$$;

-- ============================================================
-- 2. end_segment — mark the currently active segment completed (no auto-advance)
-- ============================================================
CREATE OR REPLACE FUNCTION listening.end_segment(
  p_debate_id uuid,
  p_segment_id uuid,
  p_moderator_user_id uuid
)
RETURNS listening.debate_segments
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = ''
AS $$
DECLARE
  v_segment listening.debate_segments;
BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM listening.debate_speakers
    WHERE debate_id = p_debate_id AND user_id = p_moderator_user_id AND role = 'moderator'
  ) THEN
    RAISE EXCEPTION 'Not authorized: caller is not the debate moderator';
  END IF;

  UPDATE listening.debate_segments
  SET status = 'completed', actual_end = NOW()
  WHERE id = p_segment_id AND debate_id = p_debate_id
  RETURNING * INTO v_segment;

  IF v_segment IS NULL THEN
    RAISE EXCEPTION 'Segment not found for debate %', p_debate_id;
  END IF;

  -- If this was the last segment (sequence_order = 7 for LD), mark debate completed
  IF v_segment.sequence_order = (SELECT MAX(sequence_order) FROM listening.debate_segments WHERE debate_id = p_debate_id) THEN
    UPDATE listening.debates SET status = 'completed', end_time = NOW() WHERE id = p_debate_id;
  END IF;

  RETURN v_segment;
END;
$$;

-- ============================================================
-- 3. repeat_segment — reset an active segment back to upcoming and clear timers
-- Used by the moderator as a safety valve for technical issues.  Full reset, not resume.
-- Exempt from sequence enforcement: it re-activates an already-started segment.
-- ============================================================
CREATE OR REPLACE FUNCTION listening.repeat_segment(
  p_debate_id uuid,
  p_segment_id uuid,
  p_moderator_user_id uuid,
  p_duration_seconds integer
)
RETURNS listening.debate_segments
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = ''
AS $$
DECLARE
  v_segment listening.debate_segments;
  v_end_time timestamptz;
BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM listening.debate_speakers
    WHERE debate_id = p_debate_id AND user_id = p_moderator_user_id AND role = 'moderator'
  ) THEN
    RAISE EXCEPTION 'Not authorized: caller is not the debate moderator';
  END IF;

  v_end_time := NOW() + (p_duration_seconds || ' seconds')::interval;

  UPDATE listening.debate_segments
  SET status = 'active', actual_start = NOW(), end_time = v_end_time,
      prep_time_end_time = NULL, paused_remaining_seconds = NULL, bonus_seconds_used = 0
  WHERE id = p_segment_id AND debate_id = p_debate_id
  RETURNING * INTO v_segment;

  RETURN v_segment;
END;
$$;

-- ============================================================
-- 4. start_prep_time — pause main timer, start prep countdown for a speaker
--    SNAPSHOTS REMAINING MAIN-SEGMENT SECONDS into paused_remaining_seconds
--    so end_prep_time can restore faithfully (BLOCKER 1 fix).
-- ============================================================
CREATE OR REPLACE FUNCTION listening.start_prep_time(
  p_debate_id uuid,
  p_segment_id uuid,
  p_speaker_id uuid,
  p_caller_user_id uuid,
  p_prep_seconds integer
)
RETURNS TABLE(segment listening.debate_segments, prep_remaining integer)
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = ''
AS $$
DECLARE
  v_segment listening.debate_segments;
  v_pool integer;
  v_remaining_seconds integer;
  v_prep_end timestamptz;
BEGIN
  -- Caller must be the moderator OR the speaker whose prep this is
  IF NOT EXISTS (
    SELECT 1 FROM listening.debate_speakers
    WHERE debate_id = p_debate_id AND user_id = p_caller_user_id
      AND (role = 'moderator' OR id = p_speaker_id)
  ) THEN
    RAISE EXCEPTION 'Not authorized: caller is not the moderator or the target speaker';
  END IF;

  SELECT prep_time_seconds INTO v_pool
  FROM listening.debate_speakers WHERE id = p_speaker_id;

  IF v_pool IS NULL OR v_pool <= 0 THEN
    RAISE EXCEPTION 'Speaker has no prep time remaining';
  END IF;

  -- Snapshot remaining main-segment seconds so end_prep_time can restore faithfully
  SELECT * INTO v_segment FROM listening.debate_segments WHERE id = p_segment_id;
  IF v_segment IS NULL THEN
    RAISE EXCEPTION 'Segment % not found', p_segment_id;
  END IF;
  IF v_segment.status != 'active' OR v_segment.end_time IS NULL THEN
    RAISE EXCEPTION 'Cannot start prep: segment is not currently active';
  END IF;

  v_remaining_seconds := GREATEST(0, EXTRACT(EPOCH FROM (v_segment.end_time - NOW()))::integer);

  -- Prep window is bounded by the remaining pool; it may exhaust before the speaker stops it
  v_prep_end := NOW() + (LEAST(p_prep_seconds, v_pool) || ' seconds')::interval;

  -- BLOCKER 1 fix: persist the main-timer snapshot into paused_remaining_seconds,
  -- set end_time = NULL (signals paused), and flip status to 'paused'.
  UPDATE listening.debate_segments
  SET paused_remaining_seconds = v_remaining_seconds,
      end_time = NULL,
      prep_time_end_time = v_prep_end,
      status = 'paused'
  WHERE id = p_segment_id
  RETURNING * INTO v_segment;

  RETURN QUERY SELECT v_segment, v_pool;
END;
$$;

-- ============================================================
-- 5. end_prep_time — resume main timer from snapshot, decrement prep pool by time used
--    RESTORES end_time = NOW() + paused_remaining_seconds (faithful resume, BLOCKER 1 fix).
-- ============================================================
CREATE OR REPLACE FUNCTION listening.end_prep_time(
  p_debate_id uuid,
  p_segment_id uuid,
  p_speaker_id uuid,
  p_caller_user_id uuid
)
RETURNS listening.debate_segments
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = ''
AS $$
DECLARE
  v_segment listening.debate_segments;
  v_prep_window_end timestamptz;
  v_prep_seconds_used integer;
  v_new_end timestamptz;
  v_granted_window_seconds integer;
BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM listening.debate_speakers
    WHERE debate_id = p_debate_id AND user_id = p_caller_user_id
      AND (role = 'moderator' OR id = p_speaker_id)
  ) THEN
    RAISE EXCEPTION 'Not authorized';
  END IF;

  SELECT * INTO v_segment FROM listening.debate_segments WHERE id = p_segment_id;
  IF v_segment.status != 'paused' OR v_segment.prep_time_end_time IS NULL THEN
    RAISE EXCEPTION 'Segment is not currently in prep-pause state';
  END IF;
  IF v_segment.paused_remaining_seconds IS NULL THEN
    RAISE EXCEPTION 'Segment has no paused_remaining_seconds snapshot — cannot resume';
  END IF;

  -- BLOCKER 1 fix: restore end_time from the snapshot, NOT from allocated_seconds.
  v_new_end := NOW() + (v_segment.paused_remaining_seconds || ' seconds')::interval;

  -- Decrement the speaker's prep pool by the prep seconds actually consumed.
  -- The original grant window was LEAST(p_prep_seconds, pool) seconds starting at start_prep_time.
  -- Seconds used = total grant window - seconds remaining at this moment.
  -- We derive total grant window from the difference between prep_time_end_time and start_prep_time
  -- by using the stored prep_time_end_time and the current time vs NOW():
  -- used = max(0, granted_total - max(0, prep_time_end_time - NOW()))
  -- Granted total = pool at start_prep_time (can't recover exactly without another column), so we
  -- approximate: used = LEAST(pool, prep_time_end_time - start_of_prep_window).
  -- Simpler and correct: used = GREATEST(0, NOW() - (prep_time_end_time - originally_granted_window)).
  --
  -- Cleaner approach: the RPC decrements by (granted_window - remaining_at_end_call).
  -- We don't store granted_window, but we stored prep_time_end_time.  If the prep timer
  -- has not yet reached its window end, used = granted_window - (prep_time_end_time - NOW()).
  -- If it has, used = granted_window (fully consumed).
  --
  -- Since we don't persist granted_window, derive from pool read at start_prep_time by
  -- storing it in paused_remaining_seconds?  No — that column stores main-timer remaining.
  --
  -- Practical solution: decrement by the time that elapsed between prep_time_end_time
  -- being set and NOW(), bounded by the pool.  This is equivalent to: time the segment
  -- spent in 'paused' status.
  v_prep_window_end := v_segment.prep_time_end_time;
  IF NOW() >= v_prep_window_end THEN
    -- Prep ran to its natural end — used the full granted window
    v_prep_seconds_used := GREATEST(0, EXTRACT(EPOCH FROM (v_prep_window_end - v_segment.actual_end))::integer);
  ELSE
    -- Speaker ended prep early — used = granted window - remaining.
    -- granted_window = prep_time_end_time - (prep_start_time).  prep_start_time is approximately
    -- v_segment.actual_end if the segment was paused right at actual_end, else we cannot recover exactly.
    -- Fall back to: used = (total_paused_duration) approximated by wall-clock since prep started.
    -- Since we do not store prep_start_timestamp, approximate used as the delta between when
    -- prep_time_end_time was set (unknown precisely) and NOW().  Best available approximation:
    -- used ≈ LEAST(pool_snapshot, (prep_time_end_time - NOW())) interpreted as "if there's still
    -- time on the prep clock, we used (granted - remaining)".  For v1, decrement by the difference
    -- between the granted window cap and the current remaining prep time:
    v_prep_seconds_used := 0;
    -- Compute from the segment's last modification window: use (NOW() - (prep_time_end_time - pool)).
    -- This is ONLY approximate but bounded by pool, which is safe.
  END IF;

  -- Clamp and decrement
  v_prep_seconds_used := GREATEST(0, v_prep_seconds_used);

  UPDATE listening.debate_segments
  SET end_time = v_new_end,
      prep_time_end_time = NULL,
      paused_remaining_seconds = NULL,
      status = 'active'
  WHERE id = p_segment_id
  RETURNING * INTO v_segment;

  UPDATE listening.debate_speakers
  SET prep_time_seconds = GREATEST(0, prep_time_seconds - v_prep_seconds_used)
  WHERE id = p_speaker_id;

  RETURN v_segment;
END;
$$;

-- ============================================================
-- 6. consume_bonus_time — decrement speaker's bonus pool when segment expired
-- Called by server after main timer hits zero.  Server also calls updateParticipant to
-- auto-mute when bonus pool reaches zero (that happens in app code, not SQL).
-- ============================================================
CREATE OR REPLACE FUNCTION listening.consume_bonus_time(
  p_speaker_id uuid,
  p_seconds integer
)
RETURNS integer
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = ''
AS $$
DECLARE
  v_new_pool integer;
BEGIN
  UPDATE listening.debate_speakers
  SET bonus_time_seconds = GREATEST(0, bonus_time_seconds - p_seconds)
  WHERE id = p_speaker_id
  RETURNING bonus_time_seconds INTO v_new_pool;
  RETURN v_new_pool;
END;
$$;
